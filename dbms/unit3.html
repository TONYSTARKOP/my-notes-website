Introduction to Relational Model and SQL

The relational model, introduced by **E.F. Codd in 1970**, revolutionized the way data is managed. Think of it as the database equivalent of Marvel introducing a cinematic universe—everything structured, interconnected, and primed for maximum utility.

Example: Imagine a relational database as a collection of Google Sheets or Excel files:
  • Each sheet: Represents a Table.
  • Rows: Are individual data records (like a student’s details).
  • Columns: Represent attributes of those records (e.g., Name, Age).

Example Table: Students

StudentID Name Age
1 John 20
2 Mary 22

Core Concepts of the Relational Model

1. Relations (Tables)

Tables contain rows (tuples) and columns (attributes).

  • Attributes: Column names (e.g., Name, Age).
  • Tuples: Rows of data (e.g., [John, 25]).
Example: In a company’s database:
  • Table: Employees
  • Attributes: EmployeeID, Name, Department
  • Tuple: [101, Alice, HR]

2. Keys

Keys ensure uniqueness and establish relationships between tables.

  • Primary Key: A unique identifier for rows (e.g., EmployeeID).
  • Foreign Key: Links tables by referencing a primary key in another table.
Example:

Consider two tables:

Employees Departments
EmployeeID DeptID
101 1
102 2

Primary Key: EmployeeID (unique for each employee). Foreign Key: DeptID in the Employees table references DeptID in the Departments table.

Relational Algebra

Relational Algebra is a procedural query language that defines operations on relations (tables). Think of it as SQL’s ancient, nerdier sibling who doesn’t get enough credit but is still critical for database theory.

Key Operations:

  • Select (σ): Filters rows based on conditions.
  • Project (π): Selects specific columns.
  • Join (⋈): Combines rows from two tables.
Example:

Consider a table Students:

StudentID Name Age
1 John 20
2 Mary 22
  • Select: σ(Age > 20) retrieves:
    • [2, Mary, 22]
  • Project: π(Name) retrieves:
    • [John, Mary]

SQL Basics

1. Creating Tables

        CREATE TABLE Students (
            StudentID INT PRIMARY KEY,
            Name VARCHAR(50),
            Age INT
        );

2. Inserting Data

        INSERT INTO Students (StudentID, Name, Age)
        VALUES (1, 'John', 20);
