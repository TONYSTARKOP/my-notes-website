Unit 2: Entity-Relationship Model (ER Model)

What is the Entity-Relationship Model?

The Entity-Relationship (ER) Model is a high-level conceptual data model that helps in designing databases by visually representing the data and the relationships between them. It uses entities (objects) and relationships between those entities to describe how data interacts within a database.

In simpler terms, the ER Model is like creating a blueprint for your database system. You identify the key pieces of information (entities) and then define how they interact (relationships).

Key Concepts of the ER Model

  • Entity: An entity is any object or concept about which data is stored. It can be a person, place, thing, or event.
  • Attributes: Attributes are the properties or characteristics of an entity. For example, an entity 'Employee' might have attributes like EmployeeID, Name, and DateOfBirth.
  • Entity Set: A collection of similar entities is called an entity set. For example, an 'Employee' entity set contains all employee records.
  • Relationship: A relationship describes how two or more entities are associated with each other. For example, an employee "works in" a department.
  • Relationship Set: A set of similar relationships is a relationship set. For example, the "works in" relationship between employees and departments.

ER Diagram Components

ER diagrams represent entities, attributes, and relationships using various symbols:

  • Rectangles: Represent entities.
  • Ellipses: Represent attributes.
  • Diamonds: Represent relationships.
  • Lines: Connect entities to their attributes and relationships.

Example: An ER diagram for a simple library database might include entities like 'Book' and 'Author' with a relationship 'written by'. The attributes of 'Book' could be 'BookID', 'Title', and 'PublicationYear'.

Types of Relationships

In an ER model, relationships can be of various types, defined by how entities relate to each other:

  • One-to-One (1:1): A single entity is related to a single instance of another entity.
  • One-to-Many (1:N): One entity is related to multiple instances of another entity.
  • Many-to-Many (M:N): Multiple instances of one entity are related to multiple instances of another entity.

Example: In a university database, the relationship between 'Department' and 'Professor' is 1:N, as one department can have many professors, but each professor works in only one department.

Keys in ER Model

In the ER Model, keys are used to uniquely identify an entity or a relationship:

  • Primary Key: A unique identifier for an entity. No two entities in the same set can have the same primary key.
  • Foreign Key: An attribute that creates a link between two entities. It is the primary key from one entity set that appears as an attribute in another entity set.
  • Composite Key: A primary key that consists of two or more attributes.
  • Candidate Key: A set of attributes that can uniquely identify an entity, but one of them is chosen as the primary key.
